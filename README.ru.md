# TL-Gate — TimeLayer Agent Gate

[English](README.md) · **Русский**

## Простыми словами

ИИ-агенты уже пишут файлы, дёргают API, двигают деньги. Неудобный вопрос не в
том, «умеют ли они действовать», а в том, **«кто разрешил именно это действие —
и чем вы это докажете потом?»**

Сегодня ответ обычно — строчка в логе, которую написал сам агент, или хук,
который можно выключить. И то и другое — рассказ, а не доказательство.

**TL-Gate — это ворота между намерением агента и реальным побочным эффектом.**
Прежде чем что-то произойдёт, действие обязано предъявить подписанную,
независимо проверяемую квитанцию: *этот* владелец разрешил *этому* агенту
сделать *это* действие над *этой* целью *этим* инструментом. Нет квитанции на
именно это действие — ничего не происходит. Вся идея в одной формуле:

```text
Оркестратор планирует.
Агент предлагает действие.
TL-Gate управляет допуском и проводит действие через контролируемую границу.
Verifier проверяет квитанции.
Second Brain принимает только знание, связанное с доказательством.
```

TL-Gate — **не** очередной оркестратор, не LLM и не память. У него одна
работа: обязательная fail-closed граница на квитанциях, доказуемо связывающая
разрешение, область действия, инструмент, исполнение, проверку и финальность.

## Зачем это вам

- **Вы запускаете ИИ-агентов** и хотите границу, которую нельзя «уговорить»:
  фраза модели «готово» здесь не доказывает ничего.
- **Вы отвечаете перед аудитом или EU AI Act** (ст. 12/19: защищённые от
  подделки записи действий high-risk ИИ, действует с 02.08.2026) — нужны
  свидетельства, которые переживут враждебную проверку, офлайн и без веры
  вендору на слово.
- **Вы делаете инструменты для агентов** и хотите управляемое исполнение без
  переезда на чужой фреймворк: TL-Gate нейтрален к оркестратору по построению
  (принцип P-01).

## Статус: Phase 0 — заморозка протокола

Репозиторий строится строго по [спецификации](SPEC.ru.md) (русская редакция
пока нормативная; [английская](SPEC.md) следует за ней по смыслу). Сегодня
здесь мало, но честно:

| Часть | Состояние |
|---|---|
| `tl-gate-core` — канонический `ActionIntent`, BLAKE3-коммитменты с domain separation, решения гейта, состояния цепочки | реализовано + тесты |
| `tl-gate-verifier-bridge` — только привязанная офлайн-проверка через официальный `timelayer-verifier` (`--expect` обязателен) | реализовано + тесты |
| CLI `tl-gate` — `intent digest`, `verify`, `spec-version` | реализовано |
| Всё остальное по спеке (broker, policy, validation, finalization, capsules) | **пока нет — и падает закрыто**: нереализованные команды выходят `STOP(NOT_IMPLEMENTED)`, никакого притворного ALLOW |

Попробовать:

```bash
cargo build
./target/debug/tl-gate intent digest examples/intent.json
./target/debug/tl-gate verify cert.tlcert bundle.tlbundle --expect <дайджест>
```

## Правила, которые не гнутся

1. **Нет валидной квитанции на именно это действие → нет действия.**
   Отсутствующая, просроченная, отозванная или чужая квитанция — всегда STOP.
2. **Ворота стоят до побочного эффекта**, а не после.
3. **Агент не может разрешить сам себе**, а текст модели «успешно» — не
   доказательство.
4. **Неопределённость не превращается в разрешение**: таймаут, незнакомая
   схема, недоступный verifier — всё это STOP.
5. **Только BLAKE3, с domain separation** — голые 32 байта без типа не
   коммитмент (см. аудит 2026-07-11, P0-04).
6. **Logless по умолчанию**: ограниченные неизменяемые evidence-капсулы вместо
   бесконечного редактируемого журнала. Проверка работает офлайн, против
   кворумных подписей сети — а не против нашего честного слова.

## Место среди остальных репозиториев TimeLayer

- [`timelayer-verifier`](https://github.com/TimeLayer-OS/timelayer-verifier) —
  независимый офлайн-верификатор. TL-Gate его вызывает и никогда не
  переизобретает.
- [`TL-Agent`](https://github.com/TimeLayer-OS/TL-Agent) — receipt-gated SDK
  агента; его permission-бандлы станут одним из источников квитанций через
  адаптер.
- [`timelayer-second-brain`](https://github.com/TimeLayer-OS/timelayer-second-brain) —
  база знаний на квитанциях; в неё попадают только результаты с `final_receipt`.
- [`receipt-driven-examples`](https://github.com/TimeLayer-OS/receipt-driven-examples) —
  минимальные примеры-шаблоны; справочный материал, не рантайм.
